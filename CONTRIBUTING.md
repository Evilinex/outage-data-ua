# Інструкції для учасників (Contributing Guidelines)

Дякуємо за ваш інтерес до участі в нашому проєкті! Ми раді будь-якій допомозі, будь то виправлення помилок, додавання нових функцій, покращення документації або просто пропозиції.

Цей документ містить набір інструкцій для участі в проєкті.

## Як почати

1.  **Перевірте існуючі Issues**: Перед тим як почати роботу, перегляньте список відкритих задач (Issues), щоб переконатися, що ніхто інший вже не працює над тим самим.
2.  **Створіть Issue**: Якщо ви знайшли помилку або маєте ідею для нової функції, створіть нове Issue, щоб обговорити це з командою.

## Процес розробки

1.  **Форкніть репозиторій**: Зробіть Fork цього репозиторію на свій акаунт GitHub.
2.  **Клонуйте репозиторій**: Склонуйте ваш форк на локальну машину.
3.  **Створіть гілку**: Створіть нову гілку для ваших змін. Називайте гілки змістовно, наприклад `fix/issue-123` або `feat/new-feature`.
    ```bash
    git checkout -b feat/my-new-feature
    ```
4.  **Внесіть зміни**: Напишіть код, додайте тести (якщо це можливо) та переконайтеся, що все працює коректно.
5.  **Зробіть коміт**: Зробіть коміт ваших змін з зрозумілим описом.
    ```bash
    git commit -m "Додано нову функцію для обробки даних"
    ```
6.  **Запуште зміни**: Відправте зміни у ваш форк.
    ```bash
    git push origin feat/my-new-feature
    ```
7.  **Створіть Pull Request (PR)**: Відкрийте Pull Request з вашої гілки до основної гілки (`main` або `master`) оригінального репозиторію.

## Правила оформлення коду

*   Дотримуйтесь стилю коду, який вже використовується в проєкті.
*   Пишіть зрозумілі коментарі до коду там, де це необхідно.
*   Використовуйте зрозумілі назви змінних та функцій.

## Повідомлення про помилки

Якщо ви знайшли помилку, будь ласка, створіть Issue, використовуючи шаблон для помилок (Bug Report). Надайте якомога більше деталей, щоб ми могли відтворити та виправити проблему.

## Пропозиції нових функцій

Якщо у вас є ідея для нової функції, створіть Issue, використовуючи шаблон для запиту функцій (Feature Request). Опишіть, яку проблему вирішує ця функція і як вона має працювати.

Дякуємо за вашу допомогу!
